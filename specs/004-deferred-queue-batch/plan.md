# Implementation Plan: Deferred Queue for Batch and Background Requests

**Branch**: `004-deferred-queue-batch` | **Date**: 2026-03-26 | **Spec**: [spec.md](spec.md)
**Tracking issue**: #696 | **Sub-issue**: #810 | **Depends on**: #809 (merged as #825)

## Summary

Batch and Background SLO-class requests park in a deferred queue when the cluster is busy, and are promoted to normal processing when the cluster becomes idle. This delivers "spot/batch" semantics without new configuration — deferral is unconditional for `"batch"` and `"background"` SLO classes. The change touches 3 `sim/cluster/` files for the core logic plus 2 `cmd/` files for CLI output of the new `deferred_horizon_interrupted` metric.

## Behavioral Contracts (GIVEN/WHEN/THEN)

### BC-D1: Deferral when busy

**GIVEN** a cluster with one or more instances processing requests (isBusy() == true)
**WHEN** a Batch or Background request reaches `AdmissionDecisionEvent.Execute()`
**THEN** the request is appended to `cs.deferredQueue` and the method returns without calling `admissionPolicy.Admit()`, without incrementing `cs.rejectedRequests`, and without emitting a trace record

### BC-D2: No deferral when idle

**GIVEN** a cluster where all instances have QueueDepth + BatchSize + InFlightRequests == 0 (isBusy() == false)
**WHEN** a Batch or Background request arrives
**THEN** the request proceeds through normal admission (no deferral)

### BC-D3: Promotion on idle transition

**GIVEN** `len(cs.deferredQueue) > 0` after an instance completes its last request
**WHEN** the idle-capacity check fires at the end of the event loop body
**THEN** all requests in `deferredQueue` are injected as `ClusterArrivalEvent{time: c.clock}` into `c.clusterEvents`, and `deferredQueue` is truncated to length 0

### BC-D4: Real-time requests unaffected

**GIVEN** a mixed workload with Critical + Batch requests, same seed, compared against Critical-only
**WHEN** both simulations complete
**THEN** the p50/p99 latency distributions for Critical requests are identical across both runs

### BC-D5: INV-1 conservation

**GIVEN** any simulation with Batch/Background requests and a finite horizon
**WHEN** the simulation ends
**THEN** `injected == completed + still_running + still_queued + shed + dropped_unservable + timed_out + deferred_horizon_interrupted`

### BC-D6: INV-9 — no oracle access

**GIVEN** any request in the deferral intercept
**WHEN** `isBusy()` or the intercept logic executes
**THEN** `e.request.OutputTokens` is never read

### BC-D7: Empty cluster is not busy

**GIVEN** a ClusterSimulator with zero instances
**WHEN** `isBusy()` is called
**THEN** returns false; all requests (including Batch/Background) are admitted normally

## Implementation Details

### `sim/cluster/cluster.go`

#### New field on `ClusterSimulator` struct

```go
deferredQueue []*sim.Request // Batch/Background requests awaiting idle capacity (Phase 1B-1b)
```

No initialization needed in struct literal — `nil` slice is safe for `len()` and `append()`.

#### `isBusy() bool`

```go
// isBusy returns true when any instance has non-zero effective load.
// Matches the three-component definition: QueueDepth + BatchSize + InFlightRequests > 0.
// An empty instance pool (zero instances) is not busy.
func (c *ClusterSimulator) isBusy() bool {
    for _, inst := range c.instances {
        if inst.QueueDepth()+inst.BatchSize()+c.inFlightRequests[string(inst.ID())] > 0 {
            return true
        }
    }
    return false
}
```

#### `promoteDeferred()`

```go
// promoteDeferred injects all deferred requests as ClusterArrivalEvents at the current clock.
// Called when isBusy() transitions to false. Clears deferredQueue after injection.
func (c *ClusterSimulator) promoteDeferred() {
    for _, req := range c.deferredQueue {
        heap.Push(&c.clusterEvents, clusterEventEntry{
            event: &ClusterArrivalEvent{time: c.clock, request: req},
            seqID: c.nextSeqID(),
        })
    }
    c.deferredQueue = c.deferredQueue[:0]
}
```

#### `DeferredQueueLen() int`

```go
// DeferredQueueLen returns the number of requests still in the deferred queue at simulation end.
// Panics if called before Run(). Used by cmd/ to populate RawMetrics.DeferredHorizonInterrupted.
func (c *ClusterSimulator) DeferredQueueLen() int {
    if !c.hasRun {
        panic("ClusterSimulator.DeferredQueueLen() called before Run()")
    }
    return len(c.deferredQueue)
}
```

#### Idle-capacity check in `Run()`

At the end of the main event loop body (after both the cluster-event and instance-event processing branches), before the next loop iteration:

```go
// Phase 1B-1b: promote deferred Batch/Background requests when cluster becomes idle.
if len(c.deferredQueue) > 0 && !c.isBusy() {
    c.promoteDeferred()
}
```

### `sim/cluster/cluster_event.go`

#### Pre-admission deferral intercept in `AdmissionDecisionEvent.Execute()`

Insert at the TOP of the method, before `buildRouterState()`:

```go
// Phase 1B-1b: Batch and Background requests are deferred when the cluster is busy.
// They bypass tier-shed admission entirely and re-enter as ClusterArrivalEvents on idle.
// This intercept fires BEFORE buildRouterState/Admit — deferred requests are held, not rejected.
if (e.request.SLOClass == "batch" || e.request.SLOClass == "background") && cs.isBusy() {
    cs.deferredQueue = append(cs.deferredQueue, e.request)
    return
}
```
